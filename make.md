# 사진 제출 및 검수 시스템 요구사항

## 1. 프로젝트 개요

기업별로 필요한 사진을 고객에게 요청하고, 고객은 휴대폰에서 사진을 촬영
및 제출하며, 관리자는 제출된 사진을 검수(통과/반환)할 수 있는
시스템이다.

기업마다 제출해야 하는 사진 항목이 다를 수 있으며, 관리자는 기업별로
필요한 사진 목록을 자유롭게 구성할 수 있다.

------------------------------------------------------------------------

# 2. 사용자(고객) 기능

## 2.1 접속

-   고객은 휴대폰으로 사이트에 접속한다.
-   회원가입 및 로그인은 제공하지 않는다.
-   전화번호만 입력하여 본인을 확인한다.

## 2.2 전화번호 조회

-   고객은 기업에 등록된 전화번호를 입력한다.
-   서버는 전화번호로 기업을 조회한다.
-   기업당 전화번호는 1개만 등록 가능하다.

### 조회 실패

다음과 같은 경우 접근할 수 없다.

-   등록되지 않은 전화번호
-   관리자에게 전화번호가 등록되지 않은 기업

메시지

> 등록되지 않은 전화번호입니다.

------------------------------------------------------------------------

## 2.3 제출 상태

전화번호 조회 후 현재 상태를 확인한다.

### 최초 제출

관리자가 지정한 모든 필수 사진을 표시한다.

### 검수 대기

이미 제출했지만 관리자가 아직 검수하지 않은 경우

> 현재 관리자가 사진을 확인 중입니다.

를 표시하며 추가 제출은 불가능하다.

### 반환

관리자가 일부 사진을 반환한 경우

-   반환된 사진만 표시
-   통과된 사진은 다시 제출하지 않음
-   반환 사유를 함께 표시

------------------------------------------------------------------------

## 2.4 사진 목록

각 사진 항목에는 다음 정보를 표시한다.

-   사진 명칭
-   사진 예시
-   사진 설명

반환된 사진은 추가로

-   반환 사유(강조 색상)

를 함께 표시한다.

------------------------------------------------------------------------

## 2.5 사진 업로드

각 항목마다

-   카메라 촬영
-   사진첩 선택

중 하나를 선택하여 업로드한다.

제약사항

-   사진은 항목당 1장
-   이미지 파일만 허용(JPG, PNG, HEIC 등)

------------------------------------------------------------------------

## 2.6 제출

전송 버튼은 모든 필수 사진이 업로드된 경우에만 활성화된다.

미제출 사진이 존재하면

-   해당 위치로 자동 이동
-   업로드 안내 메시지 출력

------------------------------------------------------------------------

## 2.7 제출 완료

제출 완료 후

> 감사합니다. 직원이 확인 중입니다.

를 표시한다.

동시에 관리자에게 사진 제출 알림 문자를 발송한다.

(문자 API는 별도 연동)

------------------------------------------------------------------------

# 3. 서버 처리

## 사진 저장

항상 최신 제출본만 유지한다.

실제 저장 파일명은 UUID를 사용한다.

예시

    /uploads/
     ├── 6dd99f5d-7d8f-4a9c-a3a6.jpg
     ├── 51bf54d3-4b66-49fd-b2cf.jpg

DB에는 다음 정보를 저장한다.

-   기업명
-   사진 명칭
-   UUID 파일명
-   원본 파일명
-   저장 경로
-   업로드 일시

관리자 화면에서는

    기업명_사진명

형태로 표시한다.

기존 제출 사진은 새로운 제출 사진으로 교체된다.

------------------------------------------------------------------------

# 4. 관리자 기능

## 4.1 요청 관리

### 기업 선택

기업명 또는 전화번호로 검색한다.

기업을 선택하면

-   전화번호 확인
-   전화번호 등록 및 수정 가능

기업당 전화번호는 1개만 등록 가능하다.

------------------------------------------------------------------------

### 필요 사진 지정

관리자가 등록한 요구 사진 목록 중 필요한 항목을 다중 선택한다.

저장 시 해당 기업의 제출 목록이 생성된다.

------------------------------------------------------------------------

## 4.2 요구 사진 관리

등록 항목

-   사진 명칭
-   예시 이미지
-   설명

제약사항

-   사진 명칭은 중복 불가

------------------------------------------------------------------------

## 4.3 제출 사진 조회

조회 항목

-   기업명
-   제출일시
-   사진 목록

각 사진마다

-   사진 보기
-   통과
-   반환

처리가 가능하다.

------------------------------------------------------------------------

### 통과

사진을 승인한다.

### 반환

반환 사유 입력은 필수이다.

예시

-   사진이 너무 어둡습니다.
-   인물 동작이 정확하지 않습니다.
-   건물 전체가 나오도록 다시 촬영해주세요.

반환 버튼 클릭 시

> 반환 시 기존 사진은 삭제되며 고객은 다시 촬영해야 합니다.

안내를 표시한다.

관리자가 확인하면

-   기존 사진 삭제
-   반환 상태 저장
-   반환 사유 저장

------------------------------------------------------------------------

# 5. 고객 재접속

반환된 사진만 다시 표시한다.

통과된 사진은 표시하지 않는다.

반환 사유는 사진 설명과 함께 강조하여 표시한다.

재제출 후 다시 검수 대기 상태가 된다.

------------------------------------------------------------------------

# 6. 관리자 상태 표시

기업 목록에서 현재 상태를 한눈에 확인할 수 있어야 한다.

상태

-   제출 대기
-   검수 중
-   일부 반환
-   완료

상태에 따라 색상으로 구분하는 것을 권장한다.

------------------------------------------------------------------------

# 7. 전체 흐름

``` text
관리자
    │
    ├─ 기업 등록
    ├─ 전화번호 등록
    ├─ 요구 사진 지정
    └─ 저장
          │
          ▼
고객
    │
    ├─ 전화번호 입력
    ├─ 기업 조회
    ├─ 제출 상태 확인
    ├─ 사진 업로드
    ├─ 제출
    └─ 완료
          │
          ▼
서버
    │
    ├─ UUID로 사진 저장
    ├─ DB 메타데이터 저장
    └─ 관리자 문자 발송
          │
          ▼
관리자
    │
    ├─ 제출 사진 확인
    ├─ 통과
    └─ 반환(사유 입력)
          │
          ▼
고객
    │
    └─ 반환된 사진만 재제출
```
